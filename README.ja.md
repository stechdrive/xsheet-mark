# xsheet-mark

任意のアプリの上に重ねて、ペンやマウスで注釈を書き込める Windows 用の透過オーバーレイです。下で動いているアプリのキーボード・マウス入力は妨げません。

*English: [README.md](README.md)*

## 用途の例

- **映像編集ソフトでスポッティング** — Premiere / DaVinci Resolve / VLC 等でスクラブ再生しながらタイミングをペンで書き込み、画像ごとに PSD で書き出す
- **リファレンス画像への注釈** — 画像をキャンバスにドロップして注釈・ラフを書き、書き出す
- **アニメのタイムシート作業**（原案の用途）— PSD を読み込みペンで注釈 → 元レイヤー構造を保持したまま最上位に `xsheet-mark` レイヤーを追加した PSD を書き出す
- **任意のアプリへのリアルタイム注釈** — スライド・ブラウザ・社内ツール等の上に重ねて書き込み、操作は裏のアプリへスルー

## 特徴

- **透過・最前面のオーバーレイ** — 下のアプリのキーボード・ポインター入力を妨げない
- **ペンタブ対応** — Windows Ink 経由で筆圧取得（HUION で動作確認、Wacom 系も動作見込み）
- **無限キャンバス** — 任意枚数の画像を並べて読み込み、自由にパン／ズーム
- **6 色・3 段階の線幅・ポイント消しゴム** — ペンと消しゴムそれぞれに独立した線幅記憶
- **オーバーレイのロック（クリックスルー）** — ウィンドウを「見えるが触れない」状態に切り替えて、クリックやペン入力を裏のアプリへ貫通させる。描画は見えたまま残る。解除はタスクバーのサムネイルボタンから。
- **Undo / Redo** — 描画・画像ドロップ・画像移動・キャンバスクリア
- **ウィンドウと画像の不透明度をそれぞれ独立調整**
- **対応画像形式** — PSD、PSB、JPG、PNG、BMP、GIF、TIFF、TGA
- **PSD 書き出し 2 モード**：
  - **💾 画像ごと** — キャンバス上の各画像を個別の PSD に。PSD 入力は元のレイヤー構造を保持して `xsheet-mark` レイヤーを最上位に追加、他形式は画像 + ink の新規 PSD を生成
  - **📷 ビューポートキャプチャ** — 今のビューポートに見えているものを 1 枚の PSD として書き出し（背景透明）、注釈を他の素材に合成する用途向け
- **画像移動でメモも追従** — 画像矩形内に描いたストロークは画像と一緒に移動
- **キャンバスリセット** — 1 クリックで全消去、取り消し可
- **ウィンドウサイズ・位置・不透明度をセッション間で記憶**
- **日英 UI** — Windows 表示言語から自動判定

## 動作環境

- Windows 10 または 11（64 ビット）
- 筆圧を使う場合、ペンタブドライバの設定で **「Windows Ink」を有効** にする必要あり（マウスだけなら不要）

## 使い始め方

1. [Releases](https://github.com/stechdrive/xsheet-mark/releases) から `xsheet-mark-v*.zip` をダウンロード
2. 展開 → `xsheet-mark/` フォルダに `XsheetMark.exe` + WPF ネイティブ DLL が入っている
3. そのフォルダ内の `XsheetMark.exe` を実行（同フォルダの DLL が必要）
4. 画像をウィンドウにドロップ（または画像なしで描画）→ 注釈 → 💾 または 📷 で書き出し

## 使い方

### ウィンドウ周り

- **上部の黒いバー** — ドラッグでウィンドウを移動
- **—（最小化）** — タスクバーに収納。タスクバーアイコンをクリックで復帰
- **✕（閉じる）** — アプリ終了
- **ウィンドウの縁** — カーソルを近づけるとリサイズカーソルに変化、ドラッグでリサイズ
- **不透明度パネル（右上）** — 2 本のスライダー：
  - *ウィンドウ*（0.2〜1.0）：オーバーレイ全体を透かして下のアプリを見る
  - *画像*（0.0〜1.0）：読み込んだ画像を一括で薄くする（ink はそのまま見える）

### ツールバー（左側）

| アイコン | 操作 |
|---|---|
| ↶ | 直前の描画／消し／画像ドロップ／画像移動／リセットを取り消し |
| ↷ | やり直す |
| ✎ | ペン |
| ⌫ | 消しゴム（ストローク全体ではなく、触れた場所だけを削る。サイズは線幅選択と連動） |
| ✥ | 画像移動 — 画像をドラッグすると、その画像矩形内のストロークも一緒に移動 |
| ● | 現在のインクの色 — **長押し**（約 0.35 秒）で色選択ポップアップを開く（黒／白／赤／青／緑／黄） |
| ▬ | 線幅（ペン・消しゴムそれぞれが独立に記憶） |
| ⛶ | 全画像がウィンドウに収まるようフィット |
| 💾 | 画像ごとに PSD で書き出し |
| 📷 | 現在のビューポートを 1 枚の PSD としてキャプチャ（背景透明） |
| 🔒 | オーバーレイをロック — ウィンドウをクリックスルー化して入力を裏のアプリへ流す。[オーバーレイのロック](#オーバーレイのロッククリックスルー) 参照 |
| 🗑 | キャンバスをクリア（取り消し可） |

### キャンバス操作

- **ホイール** — カーソル位置を中心にズーム
- **中クリックドラッグ** または **右クリックドラッグ** — キャンバスをパン
- 画像ファイルをウィンドウにドラッグ＆ドロップで読み込み
- **画像を読み込まなくても**キャンバスに直接描画可能

### 書き出し

ボタン 2 つで使い分け：

- 💾 **Export** — キャンバス上の画像ごとに `<元ファイル名>-marked.psd` を 1 つずつ出力。PSD 入力は元のレイヤー構造を保ったまま、サムネイルと composite preview（Maximize Compatibility の領域）も書き直すのでファイラーのプレビューも注釈入りになる。キャンバスに画像が無い場合はビューポートサイズの 1 枚 PSD にフォールバック
- 📷 **Capture** — 画像の有無に関わらず、現在のビューポートを **背景透明**の 1 枚 PSD に。「透過注釈を撮って後で素材に合成する」用途向け

どちらもフォルダ選択で出力先を指定、ファイル名が衝突した場合は `-2, -3...` の連番で回避。

### オーバーレイのロック（クリックスルー）

ツールバーの **🔒** をクリックするとオーバーレイがクリックスルー状態になります。描画は画面上に残ったまま、クリックやペン入力は裏のアプリへ貫通するので、メモを表示したまま裏のアプリでそのまま作業を再開できます。

ロック中はウィンドウ自体がクリックを受け付けないので、解除は**ウィンドウの外**から行います：

- タスクバーの xsheet-mark アイコンにマウスを重ねてサムネイルプレビューを表示 → その下に出る **🔓** ボタンをクリック。

ロック中はボーダーが赤くなるので、状態が一目で分かります。初回のロック時には解除方法を説明する警告ダイアログが出ます。チェックボックスで次回以降非表示にできます。

### ペン筆圧

ペンタブドライバ側の設定で「Windows Ink」を有効にしてください。WPF は Windows Ink／Real-Time Stylus 経由で筆圧を読み取ります。WinTab のみの構成では筆圧が渡りません。
